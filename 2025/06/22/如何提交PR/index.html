如何提交PR

引言

今天，我第一次提交的PR通过审核，第一次成为了开源项目Hexo Pro的贡献者！

激动之余，我想总结一下目前的经验，描述我现在是如何在Github上进行协作，贡献代码的。

何为Github

写过代码的都知道，软件工程是一件很繁琐的事情，需要大家进行协作。协作会带来很大的问题：今天你改动一次，明天我改动一次，改动还可能重叠。这可怎么办呢？

Git，作为一个项目协作软件，可以很好地解决这个问题，它通过创造分支，以及对代码分支的各项操作实现良好的版本管理，从而解决协作问题。

读到这，你是否有一种想法，把Git放在互联网上，这样任何人都可以参加我的项目，项目的进度是不是会变得很快呢？

Github就是这样的平台，它是全球最大的开源平台，任何人都可以将项目托管到这里，并且可以让任何其它人来参与此项目的建设。

Github的主要流程框架

基于我个人的浅薄理解，Github管理项目的主要流程框架包括：

  1. 创建仓库（项目）：新建项目仓库，生成远程代码存储地址。
  2. 上传初始代码到仓库：将本地初始代码推送到远程仓库，完成首次代码同步。
  3. 协作者从仓库里下载代码（拉取代码）：协作者通过 Git 命令从远程仓库拉取代码到本地，获取项目最新版本。
  4. 协作者在本地更新代码、上传代码到仓库：协作者在本地修改代码后，通过 Git 提交变更并推送到远程仓库，提交时附明更新说明。
  5. 项目负责人合并协作者上传的代码：负责人通过 Git 合并协作者推送的分支代码，解决冲突后将变更整合到主分支，确保项目代码统一。

可以分为三个视角看待：

首先是应用者：只需要克隆代码本地使用即可。

其次是项目负责人：需要创建代码和仓库，审核协作者上传的代码。

最后是协作者：需要拉取代码，更新代码，上传代码。

在开源项目的工作中，大部分成员都是协作者（项目负责人也可以是无需审核的协作者），在本次参与的开源工作中，我也是一个协作者，所以接下来我将具体阐述协作者是如何参与开源项目的。

参与开源项目

寻找项目

参与开源项目首先要找到合适的开源项目，Github可以搜索各种各样的项目。如何登录Github、创建Github账户，网上有很多教程不再赘述。新手可以参与一些低Star的项目，或作一些更新README、翻译等基础工作熟悉开源流程。

但是，最好要寻找一些正在积极运营的项目，有些项目早已废弃，提交代码后项目负责人不会审核，最终白费力气。同时，负责人最好有开源流程经验，具体可以看看他的主页。如果他没有经验，可能连你提交的代码都不知道如何审核。

最好的方法就是查找那些README提出欢迎PR并且最近几周有更新的项目。

这里我选择了开源项目Hexo Pro，这是一个有着100+ star的博客后台项目，我决定帮它更新README以及添加英文README。

提交代码

如果你最后一次commit觉得已经完成了所有你要做的工作，那么你可以提交到远程仓库啦！

请注意，这里并不是提交到项目负责人的仓库，只是提交到你的分支仓库

进入Github网页，点击你的分支仓库，此时顶上应该会有提示：

  • 你最近推送了一个分支，是否要发起 PR（Pull Request）

点击Compare & pull request

在跳出来的页面填写PR描述（注意礼貌和条理，论述你为什么要这样修改）

点击Create pull request

流程结束！轮到项目负责人审核，如果通过会发送邮件

总结

这是我第一次从零开始了解、接触并参与到一个真实的开源项目流程中，也第一次以协作者的身份为开源社区做出了一份贡献。整个流程虽然步骤繁琐，但每一步都非常有意义。总结下来，我认为参与开源项目的几个关键点：

  1. 项目选择很重要：选一个活跃的、有维护者的、适合新手的项目可以大大提升成功率，也能快速获得正向反馈，提升信心。

  2. 良好的沟通能力：无论是 PR 描述，还是与维护者互动，都要讲究逻辑、条理和礼貌。特别是初次参与时，良好的第一印象往往会为你打开更多机会。

  3. 坚持和反馈意识：即使第一次 PR 没有被合并，也不要气馁。你可以在 Issue 中留言、在项目主页查看项目状态，也可以继续优化自己的修改，提交更优版本。
